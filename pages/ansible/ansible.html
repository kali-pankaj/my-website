🧠 Idempotency Principle
Ansible ensures idempotency: running a playbook multiple times yields same result. Use state: present/absent effectively. Always test with --check mode.
⚡ Jinja2 & Variables
Leverage {{ ansible_os_family }}, ansible_facts. Use vars_files or group_vars for secrets. Avoid hardcoding paths.
🔐 Security First
Use ansible-vault to encrypt sensitive vars. Run playbooks with --become carefully. Limit hosts in production.
📦 Roles Structure
Roles = reusable units (tasks, handlers, defaults). Use ansible-galaxy init role_name to scaffold. Follow best-practice directory layout.

📡 Introduction to Ansible

Ansible is an open-source automation tool for configuration management, application deployment, and infrastructure as code. It's agentless, uses SSH, and YAML-based playbooks.

✨ Pro tip: Always start with ansible all -m ping -i inventory.ini to test connectivity before running playbooks.

🐚 Essential Ansible Commands

  • sudo apt update && sudo apt install ansible -y – Install Ansible on Ubuntu/Debian.
  • ansible --version – Verify installation and config paths.
  • ansible all -m ping -i inventory.ini – Test connectivity to all hosts.
  • ansible-playbook playbook.yml --syntax-check – Validate playbook before execution.
  • ansible localhost -m setup | less – Gather facts about the system (great for debugging).
  • ansible-doc -l – List all available modules.
  • ansible-galaxy install geerlingguy.nginx – Install community role.

📦 Install & Manage Services (Apache/Nginx)

---
- name: Install and Start Apache
  hosts: all
  become: yes
  tasks:
    - name: Install Apache (Ubuntu)
      apt:
        name: apache2
        state: present
    - name: Start Apache service
      service:
        name: apache2
        state: started
        enabled: yes

📂 Copy Files & Backup Management

---
- name: Copy file to remote with backup
  hosts: all
  become: yes
  tasks:
    - name: Copy file with backup
      copy:
        src: /home/user/file.txt
        dest: /tmp/file_backup.txt
        backup: yes
        owner: root
        mode: '0644'

📁 Manage Directories, Files & Permissions

---
- name: Create & Remove directory and file
  hosts: localhost
  become: yes
  tasks:
    - name: Create directory
      file:
        path: /home/ansible_test
        state: directory
        mode: '0755'
    - name: Create empty file
      file:
        path: /home/ansible_test/test.txt
        state: touch
    - name: Remove directory
      file:
        path: /home/ansible_test
        state: absent

⏰ Cron Jobs & Environment Variables

---
- name: Add cron job and env variable
  hosts: all
  become: yes
  tasks:
    - name: Add daily backup cron
      cron:
        name: "Daily backup"
        minute: "0"
        hour: "2"
        job: "/usr/local/bin/backup.sh"
    - name: Set env var in bashrc
      lineinfile:
        path: /home/user/.bashrc
        line: 'export APP_ENV="production"'
        create: yes

👥 User & Group Management (with hashed password)

---
- name: Create group, user and set password
  hosts: all
  become: yes
  tasks:
    - name: Create group
      group:
        name: devops
        state: present
    - name: Create user
      user:
        name: ansible_user
        group: devops
        shell: /bin/bash
        password: "{{ 'SecurePass123' | password_hash('sha512') }}"

🔥 Firewall Management & Tags

---
- name: Configure firewall (firewalld)
  hosts: all
  become: yes
  tasks:
    - name: Allow HTTP
      firewalld:
        service: http
        permanent: yes
        state: enabled
        immediate: yes
      tags: [ 'web' ]
    - name: Allow HTTPS
      firewalld:
        service: https
        permanent: yes
        state: enabled
        immediate: yes
      tags: [ 'web' ]

Tag usage: ansible-playbook firewall.yml --tags "web"

📌 Variables, Handlers & Conditionals

---
- name: Variables & Handlers Example
  hosts: all
  become: yes
  vars:
    web_pkg: nginx
  tasks:
    - name: Install web server
      apt:
        name: "{{ web_pkg }}"
        state: present
      notify: restart_webserver
    - name: Conditional install based on OS
      debug:
        msg: "Running on Debian family"
      when: ansible_os_family == "Debian"

  handlers:
    - name: restart_webserver
      service:
        name: "{{ web_pkg }}"
        state: restarted

🔄 Loops: Create Multiple Users & Install Packages

---
- name: Loops with users and packages
  hosts: all
  become: yes
  tasks:
    - name: Create multiple users
      user:
        name: "{{ item }}"
        state: present
        shell: /bin/bash
      loop:
        - john
        - jane
        - devops
    - name: Install packages list
      apt:
        name: "{{ item }}"
        state: present
      loop:
        - git
        - curl
        - htop
      when: ansible_os_family == "Debian"
