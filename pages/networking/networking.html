🌐 Networking for DevOps

🌐 What is a Network?

A network is a collection of connected devices such as computers, servers, mobile phones, routers, and switches that communicate with each other to share data, resources, and services.

These devices can be connected using wired connections (like Ethernet cables) or wireless connections (like Wi-Fi). Networking allows devices to exchange information quickly and efficiently.

🔑 Key Components of a Network

  • Devices: End systems like computers, laptops, servers, and mobile phones
  • Network Devices: Router (connects networks), Switch (connects devices in LAN)
  • Transmission Media: Ethernet cables, fiber optics, or wireless signals
  • Protocols: Rules like TCP/IP that define how data is transmitted

📡 Types of Networks

  • LAN (Local Area Network): Small network (home, office)
  • MAN (Metropolitan Area Network): Covers a city
  • WAN (Wide Area Network): Large network (Internet)

⚙️ How Network Works?

  • Each device has a unique identity (IP Address)
  • Data is divided into small packets
  • Packets travel through routers and switches
  • Destination device receives and reassembles data

🚀 Why Networking is Important?

  • Sharing files and resources
  • Internet access
  • Communication (email, chat, video calls)
  • Cloud computing and DevOps operations

🌍 Real-Life Example

When you open a website like google.com, your device sends a request through the network. This request travels across routers over the internet (WAN), reaches Google's server, and the response is sent back to your device.

⚙️ DevOps Use Case

  • Connecting servers in cloud environments (AWS, Azure)
  • Managing communication between microservices
  • Load balancing and traffic routing
  • Kubernetes networking (Pod-to-Pod communication)

📡 OSI Model vs TCP/IP Model

The OSI Model and TCP/IP Model describe how data moves across a network. They divide communication into layers, making it easier to understand, troubleshoot, and design systems — especially in DevOps & Cloud.

📚 OSI Model (7 Layers)

7. Application
Browser, APIs, user interaction
6. Presentation
Encryption (SSL/TLS), formatting
5. Session
Connection management
4. Transport
TCP / UDP (reliability, ports)
3. Network
IP addressing, routing
2. Data Link
MAC address, switching
1. Physical
Cables, signals, hardware

🧠 Memory Trick: All People Seem To Need Data Processing

🌐 TCP/IP Model (4 Layers)

Application
HTTP, HTTPS, FTP
Transport
TCP / UDP
Internet
IP routing
Network Access
Physical + Data Link

🔄 Layer Mapping (OSI → TCP/IP)

  • Application + Presentation + Session → Application
  • Transport → Transport
  • Network → Internet
  • Data Link + Physical → Network Access

🌍 Real-Life Flow (Opening a Website)

  1. Application: Browser sends HTTP request
  2. Presentation: Data encrypted (HTTPS)
  3. Session: Session created
  4. Transport: TCP splits data into packets
  5. Network: IP routes packets to server
  6. Data Link: MAC handles local delivery
  7. Physical: Data travels via Wi-Fi/cable

⚡ Key Differences

OSI Model TCP/IP Model
7 Layers 4 Layers
Theoretical (learning) Practical (real-world)
Strict separation Flexible structure

  • 🌐 Understanding HTTP/HTTPS & APIs

🌐 IP Address (IPv4 & IPv6)

An IP Address (Internet Protocol Address) is a unique identifier assigned to every device in a network. It works like a home address so that data can be sent and received correctly between devices.

📡 How IP Address Works?

  • Every device (computer, server, phone) has an IP address
  • When you send a request, it includes your IP (source)
  • The destination server also has an IP
  • Routers use IP addresses to deliver data packets correctly

📘 IPv4 (Internet Protocol Version 4)

IPv4 is a 32-bit address divided into 4 parts (octets). Each part ranges from 0 to 255.

Example: 192.168.1.1

  • Total addresses: ~4.3 billion
  • Format: Decimal numbers separated by dots
  • Widely used but limited (IP exhaustion)

🌐 IPv6 (Internet Protocol Version 6)

IPv6 is a 128-bit address written in hexadecimal format (0–9 and A–F).

Example: 2001:db8::1

  • Huge address space (almost unlimited)
  • No need for NAT
  • Better performance and built-in security

⚖️ IPv4 vs IPv6

  • IPv4 → 32-bit | IPv6 → 128-bit
  • IPv4 → Limited IPs | IPv6 → Massive IP space
  • IPv4 → Uses NAT | IPv6 → Direct communication

📍 Types of IP Address

  • Public IP: Accessible on the internet
  • Private IP: Used inside local networks
  • Static IP: Fixed address
  • Dynamic IP: Assigned automatically (DHCP)

🌍 Real-Life Example

When you type google.com, DNS converts it into an IP address. Your system sends a request to that IP, and the server responds back to your IP.

⚙️ DevOps Use Case

  • AWS EC2 → Public & Private IP management
  • Kubernetes → Pod IP, Service IP
  • Load Balancer → Routes traffic using IP
  • VPC → Uses CIDR blocks (IP ranges)

🌐 Public vs Private IP Address

IP addresses identify devices in a network. Understanding the difference between Public and Private IP is essential for DevOps, Cloud, and Networking.

🔓 Public IP Address

🌍 Accessible over Internet
📡 Assigned by ISP
🌐 Globally unique
☁️ Used in servers (AWS EC2, websites)
Example: 8.8.8.8

🔒 Private IP Address

🏠 Used inside local network (LAN/VPC)
🔁 Reusable across networks
🔐 Not accessible from internet
📶 Assigned by router/DHCP
Private Ranges:
  • 10.0.0.0 – 10.255.255.255
  • 172.16.0.0 – 172.31.255.255
  • 192.168.0.0 – 192.168.255.255

🔄 How NAT Works

Private IP → Router (NAT) → Public IP → Internet

👉 Your device uses a private IP, but the internet sees your public IP.

⚡ Key Differences

Public IP 🌐

  • Internet accessible
  • Globally unique
  • Assigned by ISP
  • Less secure

Private IP 🔒

  • Local network only
  • Reusable
  • Assigned by router
  • More secure

⚙️ DevOps Use Case

☁️ Public IP → Expose apps (Web servers)
🔐 Private IP → Internal communication
☸️ Kubernetes → Pod IPs are private
⚖️ LoadBalancer → Uses public IP

🧠 Easy Trick

Public = Internet 🌍 | Private = Inside Network 🏠

📊 Subnetting

Subnetting is the process of dividing a large network into smaller networks (subnets). This helps improve performance, security, and efficient IP usage.

🎯 Why Subnetting?

  • Reduces network congestion
  • Improves security (network isolation)
  • Better IP address management
  • Used in cloud (AWS VPC, Kubernetes networking)

📘 Basic Concept

An IP address has two parts:

  • Network ID: Identifies the network
  • Host ID: Identifies devices in that network

Subnetting borrows bits from the Host portion to create multiple smaller networks.

📐 CIDR Notation

CIDR (Classless Inter-Domain Routing) defines how many bits are used for the network.

Example: 192.168.1.0/24

  • /24 → 24 bits for network
  • Remaining 8 bits for hosts
  • Total IPs = 2⁸ = 256

🔢 Subnet Example

Split 192.168.1.0/24 into /26

  • /26 → 26 network bits, 6 host bits
  • Total IPs per subnet = 2⁶ = 64
  • Usable IPs = 62 (excluding network & broadcast)

📊 Subnets Created

  • 192.168.1.0 – 192.168.1.63
  • 192.168.1.64 – 192.168.1.127
  • 192.168.1.128 – 192.168.1.191
  • 192.168.1.192 – 192.168.1.255

🧠 Quick Trick (IMPORTANT)

Block Size = 256 - Subnet Mask

  • /26 → 255.255.255.192
  • Block size = 256 - 192 = 64
  • So subnets increase by 64 → 0, 64, 128, 192

⚠️ Key Points

  • First IP = Network Address
  • Last IP = Broadcast Address
  • Usable IPs = Total - 2

⚙️ DevOps Use Case

  • AWS VPC → Create public & private subnets
  • Kubernetes → Pod CIDR & Service CIDR
  • Load balancers → Route traffic across subnets
  • Security → Isolate databases in private subnet

🔗 MAC Address (Media Access Control)

A MAC Address is a unique identifier assigned to a device’s network interface card (NIC). It operates at the Data Link Layer (Layer 2) and is used for communication within a local network.

📌 What is a MAC Address?

🔢 12-digit hexadecimal address
🏷️ Unique to each device (NIC)
🔗 Works in local network (LAN)
⚙️ Used by switches for communication
Example: 00:1A:2B:3C:4D:5E

🧩 MAC Address Structure

OUI (First 6 digits) → Manufacturer
Device ID (Last 6 digits) → Unique device

👉 Example: 00:1A:2B (Vendor) + 3C:4D:5E (Device)

🔄 How MAC Works

Device A → MAC Address → Switch → MAC Table → Device B

👉 Switch uses MAC addresses to send data only to the correct device.

🔍 IP to MAC (ARP)

📡 IP Address → Logical (Layer 3)
🔗 MAC Address → Physical (Layer 2)
🔄 ARP converts IP → MAC
IP Address → ARP Request → MAC Address

📊 Types of MAC Address

Unicast

  • One-to-one communication
  • Single device

Broadcast

  • One-to-all
  • FF:FF:FF:FF:FF:FF

Multicast

  • One-to-many
  • Group communication

⚙️ DevOps Use Case

🔍 Network debugging (ARP issues)
🌐 Switch & VLAN communication
☸️ Kubernetes node networking
🔐 Security (MAC filtering)

🧠 Easy Trick

IP = Logical (Internet) 🌐 | MAC = Physical (Device) 🔗

🌐 DNS (Domain Name System)

DNS translates human-readable domain names into IP addresses. It acts like the phonebook of the internet, allowing users to access websites using names instead of numbers.

📌 What is DNS?

📖 Converts domain → IP address
🌍 Used every time you open a website
⚡ Makes internet user-friendly
Example: google.com → 142.250.x.x

🔄 How DNS Works

Browser → DNS Resolver → Root Server → TLD Server → Authoritative DNS → IP Address

👉 DNS finds the correct IP so your browser can connect to the server.

📊 Types of DNS Servers

Resolver

  • Client-side DNS query
  • Starts the request

Root Server

  • Top-level DNS
  • Directs to TLD

TLD Server

  • .com, .org, .in
  • Points to domain DNS

Authoritative

  • Stores actual records
  • Returns IP address

📁 DNS Record Types

A Record → Domain → IPv4
AAAA Record → Domain → IPv6
CNAME → Alias domain
MX Record → Mail server
TXT Record → Verification / security

⚡ DNS Caching

💾 Stores previous DNS results
⚡ Speeds up website loading
⏳ Controlled by TTL (Time To Live)

⚙️ DevOps Use Case

🌐 Configure domains (Route 53)
⚖️ Load balancing via DNS
🔐 SSL verification (TXT records)
☸️ Microservices routing

🧠 Easy Trick

DNS = Name → IP (Phonebook of Internet 📖)

📡 DHCP (Dynamic Host Configuration Protocol)

DHCP automatically assigns IP addresses and network settings to devices in a network. It removes the need for manual IP configuration and ensures smooth communication between devices.

📌 What is DHCP?

⚡ Automatically assigns IP addresses
🌐 Provides network configuration (IP, Gateway, DNS)
🖥️ Used in LAN, Wi-Fi, and Cloud
Example: Laptop gets IP like 192.168.1.10

🔄 DHCP Process (DORA)

Discover → Offer → Request → Acknowledge
DORA:
  • Discover: Client asks for IP
  • Offer: Server offers IP
  • Request: Client accepts
  • Acknowledge: Server confirms

🧩 DHCP Components

DHCP Server → Assigns IP
DHCP Client → Requests IP
IP Pool → Range of available IPs
Lease Time → Duration of IP usage

📊 Types of IP Assignment

Dynamic

  • Temporary IP
  • Changes over time

Static (Manual)

  • Fixed IP
  • Configured manually

Reserved

  • Fixed via DHCP
  • Based on MAC address

⚙️ DevOps Use Case

🌐 Auto IP assignment in VPC/LAN

🧠 Easy Trick

🌐 Routing & Switching Basics

Switching and Routing are core networking concepts. Switching connects devices within the same network (LAN), while routing connects different networks and enables communication across the internet.

🔗 Switching (Layer 2)

📡 Works inside Local Network (LAN)
🔗 Uses MAC Address
⚙️ Operates at Data Link Layer
📊 Uses MAC Table to forward data
Device A → Switch → MAC Table → Device B

👉 Switch sends data only to the correct device using MAC address.

🌍 Routing (Layer 3)

🌐 Connects different networks
📍 Uses IP Address
⚙️ Operates at Network Layer
🧭 Finds best path using routing table
Source IP → Router → Routing Table → Destination IP

👉 Router decides the best path to reach another network.

⚡ Switching vs Routing

Switching 🔗

  • Layer 2 (Data Link)
  • Uses MAC address
  • Within same network
  • Faster

Routing 🌐

  • Layer 3 (Network)
  • Uses IP address
  • Between networks
  • Path selection

📊 Types of Routing

Static Routing → Manual configuration
Dynamic Routing → Automatic (RIP, OSPF, BGP)
Default Routing → Unknown paths (internet)

⚙️ DevOps Use Case

🌐 VPC routing tables (AWS/GCP)
⚖️ Load balancer traffic flow
☸️ Kubernetes networking
🔍 Debugging network issues

🧠 Easy Trick

Switch = MAC (LAN) 🔗 | Router = IP (Internet) 🌐

🌐 NAT (Network Address Translation)

NAT allows multiple devices in a private network to share a single public IP address. It translates Private IP ↔ Public IP for communication with the internet.

📌 What is NAT?

🔄 Converts Private IP → Public IP
🌍 Enables internet access
🔐 Adds security (hides internal network)
Example: 192.168.1.10 → 49.x.x.x

🔄 How NAT Works

Private IP → Router (NAT) → Public IP → Internet

👉 Internet sees only the public IP, not your private devices.

📊 Types of NAT

Static NAT

  • 1:1 mapping
  • Fixed IP mapping

Dynamic NAT

  • Uses IP pool
  • Temporary mapping

PAT (Port Address Translation)

  • Many-to-one mapping
  • Uses port numbers

⚡ Benefits of NAT

🌐 Saves public IP addresses
🔐 Improves security
⚙️ Simplifies network management

⚙️ DevOps Use Case

☁️ NAT Gateway in cloud (AWS/GCP)
🔐 Private subnet internet access
☸️ Kubernetes outbound traffic
⚖️ Secure architecture design

🧠 Easy Trick

NAT = Private → Public Translation 🔄

🔥 Firewalls & Security Groups

Firewalls and Security Groups control network traffic by allowing or blocking connections based on rules. They are essential for network security in DevOps and Cloud.

🛡️ Firewall

🚫 Blocks or allows traffic
🌐 Works at network boundary
⚙️ Uses rules (IP, Port, Protocol)
Example: Block port 22 (SSH)
Incoming Traffic → Firewall Rules → Allow / Deny

👉 Firewall protects the system from unauthorized access.

☁️ Security Group (Cloud Firewall)

🔐 Virtual firewall (cloud)
📡 Controls inbound & outbound traffic
⚡ Works at instance level (AWS EC2)
Example: Allow HTTP (80), HTTPS (443)
User → Security Group → EC2 Instance

⚡ Firewall vs Security Group

Firewall 🛡️

  • Network-level protection
  • Can be hardware/software
  • Used in on-prem & cloud

Security Group ☁️

  • Instance-level control
  • Cloud-specific (AWS)
  • Stateful rules

📊 Rule Types

Inbound → Incoming traffic
Outbound → Outgoing traffic
Allow → Permit traffic
Deny → Block traffic

⚙️ DevOps Use Case

🔐 Secure servers (SSH access)
🌐 Allow web traffic (80 / 443)
☸️ Kubernetes network security
⚖️ Control microservices communication

🧠 Easy Trick

⚖️ Load Balancers (L4 vs L7)

A Load Balancer distributes incoming traffic across multiple servers to ensure high availability, scalability, and reliability of applications.

📌 What is a Load Balancer?

⚖️ Distributes traffic across servers
🚀 Improves performance & availability
🔁 Prevents server overload
Example: 1 app → 3 backend servers
User → Load Balancer → Server 1 / Server 2 / Server 3

🔌 Layer 4 Load Balancer (Transport Layer)

📡 Works at TCP/UDP level
⚡ Fast (no content inspection)
📍 Uses IP + Port
Example: TCP connection routing
Client → IP + Port → Server

🌐 Layer 7 Load Balancer (Application Layer)

🌐 Works at HTTP/HTTPS level
🧠 Smart routing (URL, headers)
🔐 Supports SSL termination
Example: /api → Server A, /app → Server B
Client → HTTP Request → Route by URL → Server

⚡ L4 vs L7 Load Balancer

L4 Load Balancer 🔌

  • Layer 4 (TCP/UDP)
  • Fast & simple
  • No content awareness
  • Uses IP + Port

L7 Load Balancer 🌐

  • Layer 7 (HTTP/HTTPS)
  • Smart routing
  • Content-aware
  • URL / Header based routing

📊 Load Balancing Algorithms

Round Robin → Equal distribution
Least Connections → Fewer active users
IP Hash → Same user → same server

📡 CIDR Notation (Classless Inter-Domain Routing)

CIDR is a method for allocating IP addresses and dividing networks efficiently. It replaces traditional subnetting and allows flexible network sizes using prefix length.

📌 What is CIDR?

🔢 Represents IP with prefix length
📊 Format: IP / Prefix
⚡ Efficient IP allocation
Example: 192.168.1.0/24

🧩 CIDR Structure

192.168.1.0 → Network Address
/24 → Network bits (Subnet mask)

👉 /24 means first 24 bits are network, remaining are host bits.

📊 Common CIDR Blocks

CIDR Subnet Mask Hosts
/8 255.0.0.0 16M
/16 255.255.0.0 65K
/24 255.255.255.0 256
/32 255.255.255.255 1

🔢 CIDR Calculation

Total Hosts = 2(32 - prefix)
Usable Hosts = Total - 2

👉 Example: /24 → 2⁸ = 256 → usable = 254

🌍 Real Example

192.168.1.0/24 → Range: .1 – .254 → Broadcast: .255

⚙️ DevOps Use Case

☁️ AWS VPC CIDR blocks
📡 Subnet design (public/private)
☸️ Kubernetes pod networking
🔐 Security group IP ranges

🧠 Easy Trick

/24 = Small Network 🏠 | /16 = Medium 🌆 | /8 = Large 🌍

☸️ Kubernetes Networking

Kubernetes networking enables communication between Pods, Services, and external users using a flexible and scalable networking model.

📦 Pod Networking

📡 Each Pod gets its own IP
🔗 Pods communicate directly (no NAT)
🌐 Flat network across cluster
Example: Pod A → Pod B via IP

🔌 Service Types

ClusterIP 🏠

  • Default service
  • Internal communication only

NodePort 🚪

  • Exposes service on node port
  • Accessible via NodeIP:Port

LoadBalancer ⚖️

  • External access
  • Uses cloud load balancer

🌐 Ingress Controller

🌍 HTTP/HTTPS routing
🔀 Routes based on URL/host
🔐 SSL termination
Example: /api → Service A
User → Ingress → Service → Pod

🔗 CNI (Container Network Interface)

Calico → Network + security policies
Flannel → Simple overlay network
📡 Provides Pod networking
🌐 Assigns IPs to pods
🔐 Handles network policies

🔍 Service Discovery

🌐 Uses DNS inside cluster
📛 Service name → IP
⚡ Automatic load balancing
Example: my-service.default.svc.cluster.local
Pod → DNS → Service → Pod
