AWS (Amazon Web Services)

What is AWS?

AWS is a cloud computing platform provided by Amazon. It allows users to access computing resources such as servers, storage, databases, networking, and software over the internet.

AWS allows you to rent computing resources instead of buying and maintaining physical servers.

What is Cloud Computing?

Cloud computing means using servers, storage, networking, and software over the internet instead of using your local computer or physical data center.

Why do companies use AWS?

  • No need to buy expensive hardware
  • Pay only for what you use
  • Easy to scale resources
  • High availability and reliability
  • Global infrastructure

AWS Services

IAM (Identity and Access Management) Free Global

Pricing: Free   |   Scope: Global (works across all AWS regions)

AWS IAM is a security service that controls access to AWS resources. It manages authentication (who can access) and authorization (what actions they can perform).

🔑 Core Components

  • Users: Individual identities (admin, developer)
  • Groups: Collection of users with same permissions
  • Roles: Temporary access for services/users
  • Policies: JSON-based permission rules

⚙️ Key Features

  • Fine-grained access control
  • Multi-Factor Authentication (MFA)
  • Role-based secure access
  • Works with all AWS services

🚀 Use Cases

  • Manage team access
  • Secure applications using roles
  • Provide temporary credentials
  • Apply least privilege principle

📌 Example

  • Admin: Full access
  • Developer: EC2 + S3
  • Tester: Read-only

🎯 Interview Answer

IAM is used to manage authentication and authorization in AWS using users, roles, and policies.

EC2 (Elastic Compute Cloud) Paid Regional

Pricing: Paid   |   Scope: Regional (runs inside specific AWS regions)

Amazon EC2 is a compute service that provides virtual servers (instances) in the cloud to run applications.

⚙️ Key Features

  • Launch virtual machines in minutes
  • Supports Linux and Windows
  • Pay-as-you-go pricing
  • Highly scalable and flexible
  • Secure access using SSH key pairs

📌 Example

  • Launch EC2 → Install Nginx → Host website
