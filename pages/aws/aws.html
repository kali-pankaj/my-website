AWS (Amazon Web Services)

What is AWS?

AWS is a cloud computing platform provided by Amazon. It allows users to access computing resources such as servers, storage, databases, networking, and software over the internet.

AWS allows you to rent computing resources instead of buying and maintaining physical servers.

What is Cloud Computing?

Cloud computing means using servers, storage, networking, and software over the internet instead of using your local computer or physical data center.

Why do companies use AWS?

  • No need to buy expensive hardware
  • Pay only for what you use
  • Easy to scale resources
  • High availability and reliability
  • Global infrastructure

🔐 AWS IAM (Identity and Access Management) Free Global

Pricing: Free   |   Scope: Global (works across all AWS regions)

📌 Definition

AWS IAM is a security service that controls access to AWS resources. It manages authentication (who can access) and authorization (what actions they can perform).

🏗️ Architecture

  • User: Individual identity
  • Group: Collection of users
  • Role: Temporary access for services
  • Policy: JSON permission rules
                User → IAM Policy → AWS Resource (S3 / EC2)

🔑 Core Components

  • Users: Admin, developer, tester
  • Groups: Manage multiple users
  • Roles: Used by EC2, Lambda, etc.
  • Policies: Define permissions

⚙️ Key Features

  • Fine-grained access control
  • Multi-Factor Authentication (MFA)
  • Role-based access
  • Global service
  • Secure API access

📜 IAM Policy Example

                {
                "Version": "2012-10-17",
                "Statement": [
                    {
                    "Effect": "Allow",
                    "Action": "s3:*",
                    "Resource": "*"
                    }
                ]
                }

🚀 Use Cases

  • Manage team access
  • Secure AWS services
  • Provide temporary credentials
  • Implement least privilege access

🛠️ DevOps Usage

  • Attach IAM roles to EC2
  • Secure CI/CD pipelines
  • Control Terraform access
  • Grant Kubernetes (EKS) permissions

💻 AWS CLI Example

                # Create IAM user
                aws iam create-user --user-name dev-user

                # Attach policy
                aws iam attach-user-policy \
                --user-name dev-user \
                --policy-arn arn:aws:iam::aws:policy/AmazonS3FullAccess

⚙️ Terraform Example

                resource "aws_iam_user" "dev" {
                name = "dev-user"
                }

                resource "aws_iam_user_policy_attachment" "attach" {
                user       = aws_iam_user.dev.name
                policy_arn = "arn:aws:iam::aws:policy/AmazonS3FullAccess"
                }

🔐 Security Best Practices

  • Enable MFA for all users
  • Use roles instead of access keys
  • Follow least privilege principle
  • Rotate credentials regularly

🧪 Real Example

  • Developer gets EC2 + S3 access
  • Tester gets read-only access
  • EC2 uses IAM Role to access S3

🔄 Real-World Flow

                User → IAM → Policy → AWS Resource → Access Granted / Denied

☁️ Amazon S3 (Simple Storage Service) Paid Global

Pricing: Pay-as-you-go   |   Scope: Global (data stored in selected region)

  • Key: Unique identifier (path)

                Bucket: my-app-data

                ├── logs/app.log
                └── backup/db.sql

🌍 Global vs Regional

  • Globally accessible service
  • Bucket names are globally unique
  • Data is stored in selected AWS region

Example: Bucket in ap-south-1 stores data in Mumbai but is accessible globally.

⚡ Key Features

  • Durability: 99.999999999% (11 9’s)
  • Unlimited scalability
  • High availability
  • Secure access (IAM, Policies)
  • Versioning support
  • Lifecycle automation

📦 Storage Classes

  • S3 Standard
  • Intelligent-Tiering
  • Standard-IA
  • Glacier (archive)

🎯 Use Cases

  • Static website hosting
  • Media storage
  • Backup & disaster recovery
  • Log storage
  • Big data storage

🚀 DevOps Usage

  • Terraform backend (state file storage)
  • CI/CD artifacts storage
  • Integration with CloudFront (CDN)
  • Kubernetes backups

                # Create bucket

🔐 Security

  • Bucket Policies

🔄 Real-World Flow

🌐 VPC (Virtual Private Cloud) Free Regional

Pricing: Free (pay for sub-resources like NAT, EC2)   |   Scope: Regional

📌 Definition

Amazon VPC is a networking service that allows you to create a logically isolated virtual network in AWS where you can launch resources securely.

🏗️ Architecture

                VPC (10.0.0.0/16)
                │
                ├── Public Subnet (10.0.1.0/24) → Internet Gateway
                │       └── EC2 (Web Server)
                │
                ├── Private Subnet (10.0.2.0/24) → NAT Gateway
                │       └── EC2 (App Server)
                │
                └── Private Subnet (10.0.3.0/24)
                        └── RDS (Database)

🔑 Core Components

  • VPC: Main network container
  • Subnets: Public & Private networks
  • Route Tables: Traffic routing rules
  • Internet Gateway (IGW): Internet access
  • NAT Gateway: Outbound internet for private subnet
  • Security Groups: Instance-level firewall
  • Network ACL: Subnet-level firewall

⚙️ Key Features

  • Complete network isolation
  • Custom CIDR IP range
  • Highly secure and scalable
  • Supports hybrid cloud (VPN / Direct Connect)

📐 CIDR Example

                VPC CIDR: 10.0.0.0/16

                Public Subnet:  10.0.1.0/24
                Private Subnet: 10.0.2.0/24
                DB Subnet:      10.0.3.0/24

🚀 Use Cases

  • Secure application hosting
  • 3-tier architecture (Web, App, DB)
  • Microservices networking
  • Hybrid cloud integration

🛠️ DevOps Usage

  • Deploy Kubernetes (EKS) inside VPC
  • Run EC2 instances in private subnets
  • Use ALB/NLB for traffic routing
  • Secure backend services

💻 AWS CLI Example

                # Create VPC
                aws ec2 create-vpc --cidr-block 10.0.0.0/16

                # Create Subnet
                aws ec2 create-subnet \
                --vpc-id vpc-12345 \
                --cidr-block 10.0.1.0/24

⚙️ Terraform Example

                resource "aws_vpc" "main" {
                cidr_block = "10.0.0.0/16"
                }

                resource "aws_subnet" "public" {
                vpc_id     = aws_vpc.main.id
                cidr_block = "10.0.1.0/24"
                }

🔐 Security Best Practices

  • Use private subnets for backend
  • Restrict inbound traffic using SG
  • Enable VPC Flow Logs
  • Use NAT Gateway instead of public access

🧪 Real-World Example

  • Web server in public subnet
  • App server in private subnet
  • Database in isolated subnet

🔄 Request Flow

                User → Internet → IGW → Public EC2 → Private EC2 → Database

🎤 Interview Answer

"VPC is a regional AWS networking service that provides an isolated virtual network where we can securely launch resources using subnets, route tables, and gateways."

                            ↓
                Standby DB (AZ-2) [Multi-AZ]

🧩 Supported Databases

  • PostgreSQL
  • MariaDB
  • Oracle
  • SQL Server
  • Aurora (AWS optimized)

⚙️ Key Features

  • Fully managed service
  • Automatic backups & snapshots
  • Multi-AZ high availability
  • Read replicas for scaling
  • Auto patching and maintenance

📊 RDS Deployment Types

  • Single-AZ: Basic deployment
  • Multi-AZ: High availability
  • Read Replica: Read scaling

🚀 Use Cases

  • Web application backend database
  • E-commerce platforms
  • ERP / CRM systems
  • Analytics workloads

🛠️ DevOps Usage

  • Connect RDS with EC2 / EKS applications
  • Store backend data securely
  • Use secrets manager for DB credentials
  • Automate infra using Terraform

💻 AWS CLI Example

                # Create RDS instance
                aws rds create-db-instance \
                --db-instance-identifier mydb \
                --db-instance-class db.t3.micro \

🧪 Real-World Example

  • Backend → EC2/EKS (private)
  • Database → RDS (private subnet)

🔄 Data Flow
