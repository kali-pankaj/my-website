🧠 Pipeline as Code
Always prefer Jenkinsfile over freestyle jobs. Store pipeline in SCM (Git) for versioning, review, and reusability.
⚡ Agent Strategy
Use ephemeral agents (Docker/K8s) for each build. Label agents properly: linux_agent, windows_agent to route jobs.
🔐 Credentials Security
Store SSH keys, tokens in Jenkins Credentials Store. Never hardcode secrets in pipeline. Use withCredentials block.
📦 Shared Libraries
Create shared pipeline libraries for reusable functions (email notifications, slack alerts, custom steps).

📡 Jenkins Installation & Initial Setup (Ubuntu/Debian)

  • sudo apt update && sudo apt upgrade -y – Update system packages.
  • sudo apt install openjdk-17-jdk -y – Install Java 17 (Jenkins requirement).
  • java -version – Verify Java installation.
  • sudo apt install jenkins -y – Install Jenkins LTS.
  • jenkins --version – Confirm Jenkins version.
  • sudo firewall-cmd --add-port=8080/tcp --permanent (if using firewalld) or sudo ufw allow 8080 for UFW.
  • sudo systemctl enable --now jenkins – Start Jenkins on boot.
  • sudo systemctl status jenkins – Check service status.
  • Open browser: http://localhost:8080
  • Get initial admin password: sudo cat /var/lib/jenkins/secrets/initialAdminPassword
  • Install suggested plugins, create admin user, set Jenkins URL.
✨ Pro tip: Use /etc/default/jenkins to change HTTP port if needed (e.g., HTTP_PORT=9090).

Step 1: Dashboard → New Item
Step 2: Enter Job Name: first-job | Freestyle Project → OK
Step 3: Description: "My first Jenkins automation test"

   date

   echo "Build successful!"

🔐 3. Role-Based Strategy (Authorization)

Install "Role-based Strategy" plugin
Manage Jenkins → Security → Authorization → Role-Based Strategy
Manage Roles:
  - Global roles: admin (full access), read-only (overall read)
  - Project roles: dev-team (pattern: dev-.*) with build/cancel permissions.
Assign roles to users/groups for fine-grained access control.

SCM: Git
Project Repository: https://github.com/org/shared-pipeline-lib.git
Usage: @Library('shared-library') _ at top of Jenkinsfile

📜 Declarative Pipeline Examples (Jenkinsfile)

pipeline {
    agent any
    stages {
        stage('Checkout') {
            steps {
                git 'https://github.com/example/app.git'
            }
        }
        stage('Build') {
            steps {
                sh 'mvn clean compile'
            }
        }
        stage('Test') {
            steps {
                sh 'mvn test'
            }
            post {
                always {
                    junit 'target/surefire-reports/*.xml'
                }
            }
        }
        stage('Deploy') {
            when { branch 'main' }
            steps {
                echo 'Deploying to production server'
            }
        }
    }
    post {
        failure {
            emailext subject: "Pipeline Failed", body: "Job ${env.JOB_NAME}", to: "team@example.com"
        }
    }
}

🧪 SSH Agent Pipeline with Credentials

pipeline {
    agent { label 'linux_worker' }
    environment {
        APP_HOME = '/opt/app'
    }
    stages {
        stage('Deploy via SSH') {
            steps {
                sshagent(['deploy-ssh-key']) {
                    sh 'scp target/app.jar user@server:/opt/app/'
                    sh 'ssh user@server "systemctl restart myapp"'
                }
            }
        }
    }
}

🔐 Jenkins Security & Role Management

  • Disable script security in sandbox? Only for trusted users. Use Approval for scripts.
  • Matrix-based security: Fine-grained project-level permissions.
  • Credentials binding: Use withCredentials([string(credentialsId: 'api-token', variable: 'TOKEN')]) in pipelines.
  • Audit logging: Enable "Job Configuration History" plugin to track changes.
  • Backup Jenkins home: Archive /var/lib/jenkins regularly (config, jobs, workspaces).
🔐 Always run Jenkins behind reverse proxy (nginx/Apache) with HTTPS enabled. Use Jenkins CLI safely with RBAC.
