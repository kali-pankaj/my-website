📘 1. Fundamental Git Commands (Daily Driver)

CommandDescription & Real-world UsageCopy
git initInitialize empty repository. Use --bare for remote central repo, --initial-branch=main to set default branch
git clone --depth 1 <url>Shallow clone (faster CI/CD). Saves bandwidth, only latest commit. Add --single-branch to clone only default branch
git status -sbShort branch status with upstream tracking. Shows ahead/behind counts. Perfect for daily standups
git add -AStage all changes including deletions. Safer than 'git add .' in subdirectories. Use -p for interactive staging
git commit --amend -m "new msg"Edit last commit message. Add --no-edit to add staged changes without changing message. Use --author to fix author info
git commit -am "msg"Stage tracked files + commit in one command. Won't add untracked files. Great for quick fixes on known files
git mv old newMove/rename file while preserving history. Better than manual mv + add/rm
git rm --cached fileStop tracking file but keep locally (add to .gitignore after). Perfect for removing accidentally committed secrets

🌿 Branching & Merging (GitHub Flow, GitFlow)

CommandDescription & Real-world UsageCopy
git branch -vvShow tracking branches with ahead/behind counts. Essential before pull/push to avoid conflicts
git push origin --delete feature/loginDelete remote branch after merge (clean up). Use -d for local deletion
git merge --squash featureSquash all feature commits into one before merging. Keeps main branch history clean
git merge --no-ff featureForce merge commit even if fast-forward possible. Preserves feature branch context
git branch -m old-name new-nameRename branch locally. Then git push origin -u new-name and delete old remote
git branch --sort=-committerdateList branches sorted by most recent commit. Shows active branches first
git branch --merged | grep -v "\*" | xargs git branch -dDelete all merged branches at once. Cleanup after release

🌐 Remote & Collaboration (Team Workflows)

CommandDescription & Real-world UsageCopy
git remote prune originRemove stale remote tracking branches. Run after teams delete remote branches
git push --force-with-leaseSafe force push (checks remote hasn't changed). Never use --force on shared branches
git fetch --all --pruneFetch all remotes and clean stale refs. Sync team changes without merging
git pull --rebase --autostashRebase instead of merge, auto-stash dirty working directory. Clean linear history
git remote set-url origin new-urlChange remote URL (repo moved or switched from HTTPS to SSH)
git push -u origin HEADPush current branch and set upstream. One command to start tracking a new remote branch
git config --global pull.rebase trueSet rebase as default for git pull. Prevents merge clutter globally

⏪ Undo, Reset & Recovery Mastery

CommandDescription & Real-world UsageCopy
git restore --source=HEAD~2 file.txtRestore file from 2 commits ago. Better than checkout for file recovery
git revert -m 1 <merge-commit>Revert a merge commit (specify parent line). Safe for public branches
git reset --soft HEAD~1Undo commit, keep changes staged. Perfect for fixing last commit before push
git reset --mixed HEAD~1Undo commit + unstage. Default reset behavior
git reset --hard ORIG_HEADUndo a destructive reset or merge. ORIG_HEAD saved before dangerous operations
git checkout HEAD~3 -- file.txtRestore file from 3 commits ago (older syntax, still works)

📦 Stash (Context Switching Like a Pro)

CommandDescription & Real-world UsageCopy
git stash -uStash including untracked files. Use -a to include ignored files too
git stash push -m "WIP auth" -- src/auth/Stash specific directory with message. Perfect for partial work
git stash show -p stash@{0}View diff of stashed changes before applying
git stash branch new-branch stash@{1}Create branch from stash. Recover work that drifted from base
git stash drop $(git stash list | grep "WIP" | head -1 | cut -d: -f1)Drop first stash matching pattern. Cleanup old WIPs

🏷️ Tagging & Release Engineering

CommandDescription & Real-world UsageCopy
git tag -a v2.0 <commit> -m "release notes"Annotated tag with message (recommended vs lightweight). Use for releases
git push origin --tags --forceOverwrite remote tags (⚠️ use only when necessary). Better to delete+recreate

🔍 Diff & Log Inspection (Debugging)

CommandDescription & Real-world UsageCopy
git log --since="2 weeks ago" --onelineFilter log by date. Also --until, --author, --grep for flexible search
git diff HEAD~3..HEAD --statSummary of changes in last 3 commits. Add --name-only for just file list
git log --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset'Custom pretty log with colors. Perfect for daily review
git diff branch1..branch2 -- file.txtCompare specific file between branches
git log -S "function_name" --sourceSearch commits where function_name was added/removed (pickaxe)
git blame -L 10,20 file.txtSee who changed lines 10-20. Add -C to detect moved code

🔗 Submodules (Microservices & Shared Libs)

CommandDescription & Real-world UsageCopy
git submodule add -b main <url> libs/sharedAdd submodule tracking specific branch. Prevents detached HEAD
git submodule update --init --recursive --remoteUpdate all submodules to latest commit on tracked branch
git submodule status --recursiveShow commit hashes of all submodules. Debug version mismatches
git submodule foreach 'git checkout main && git pull'Run command in each submodule. Batch updates
git submodule deinit -f -- libs/sharedRemove submodule completely (also delete .git/modules and stage deletion)
git config --global submodule.recurse trueAutomatically recurse into submodules for commands like git pull, git diff

🌳 Git Worktree (Parallel Development)

CommandDescription & Real-world UsageCopy
git worktree add -b hotfix ../project-hotfix mainCreate new worktree with new branch. Hotfix without stashing
git worktree list --porcelainMachine-readable worktree list (for scripting)
git worktree remove ../project-hotfix --forceForce remove worktree even if dirty. Use with git worktree prune after
git worktree add --detach ../temp-branch HEAD~3Create worktree at detached state (inspect old commit without branch)

🪝 Git Hooks (Automation & Quality Gates)

Hook Path / CommandDescription & Real-world UsageCopy
.git/hooks/pre-commitRun linters, formatters, secrets scan before commit. Return non-zero to abort
.git/hooks/commit-msgValidate commit message format (Conventional Commits, Jira ticket ID)
.git/hooks/pre-pushRun tests before push. Stops breaking the build
.git/hooks/pre-receive (server-side)Central hook to enforce branch policies, block force-pushes on main
git config --global core.hooksPath ~/.git-hooksShare hooks across all repositories. Version control your hooks
git hooks --list (Git 2.36+)List available hooks in current Git version

📜 Reflog – Ultimate Recovery Tool

CommandDescription & Real-world UsageCopy
git reflog --date=localShow reflog with local timestamps. Find work from specific date
git reset --hard HEAD@{2.minutes.ago}Restore state from 2 minutes ago. Natural language access
git reflog expire --expire=30.days --allKeep reflog for 30 days (default). Reduce .git size
git show HEAD@{2025-12-01 09:00:00}View commit from specific datetime. Great for audits
git reflog show --allShow reflog for all refs (branches, stash, etc.)

💾 Git LFS (Large File Storage for ML, Games, Assets)

CommandDescription & Real-world UsageCopy
git lfs track "*.psd" --lockableTrack PSD files with file locking. Prevent concurrent edits
git lfs migrate import --include="*.zip" --everythingMigrate ALL branches and tags to LFS. Rewrites history
git lfs ls-files --all --longList all LFS files with sizes. Find large assets to clean
git lfs prune --verify-remoteDelete old LFS files, check if remote still has them
git lfs fetch --all --recentDownload all LFS files (not just current checkout)
git lfs statusShow which LFS files are modified locally

⚡ Advanced Power Tools (History Rewriting, Bisect, Grep)

CommandDescription & Real-world UsageCopy

🚀 DevOps Patterns & Production CI/CD Integration

git log -1 --pretty=%BGet last commit message. Extract release notes in CD pipeline

git ls-tree -r main --name-only | xargs wc -lCount lines of code in repository. DevOps metrics

⚙️ Global Config & Time-Saving Aliases

Command / AliasDescription & Real-world UsageCopy
git config --global alias.lg "log --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit"Pretty log alias. Run 'git lg' for beautiful history
git config --global alias.unstage 'reset HEAD --'Unstage alias. Run 'git unstage file.txt'

git config --global alias.undo 'reset --soft HEAD^'Undo last commit but keep changes. Safer alternative
git config --global core.excludesfile ~/.gitignore_globalGlobal gitignore (OS-specific: .DS_Store, Thumbs.db)
git config --global init.defaultBranch mainSet default branch name to 'main' for all new repos
