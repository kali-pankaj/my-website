What is Docker

  • Docker is an open-source platform that allows developers to automate the deployment, scaling, and management of applications inside lightweight, portable containers. A container is an isolated, self-contained environment that includes everything an application needs to run: code, libraries, dependencies, and system tools.

1. Containers

  • What are Containers: Containers are a way to package an application and its dependencies in a consistent environment, regardless of the host system. They ensure that an application works the same way in any environment (development, staging, production) because the environment is containerized.
  • Isolation and Portability: Containers are isolated from the host system, but share the same OS kernel, making them lightweight compared to traditional virtual machines (VMs). This isolation means they won’t interfere with other applications running on the same system. Containers can run on any system that has Docker installed, making them highly portable.

2. Dockerfile

  • A Dockerfile is a script that contains a series of instructions on how to build a Docker image. It specifies things like:
    • Which base image to use
    • How to copy your application code into the image
    • How to install dependencies
    • What commands to run when the container starts
    • Example Dockerfile:
    
    FROM node:14
    WORKDIR /app
    COPY . .
    RUN npm install
    EXPOSE 3000
    CMD ["npm", "start"]

3. Images

  • What is Images: A Docker image is a read-only template that contains the code, runtime, libraries, environment variables, and configuration files needed to run an application. Images are used to create containers. Docker images can be built from a Dockerfile, which contains a set of instructions on how to assemble the image.
  • Base Image an Custom Image: You can start with a base image (like Ubuntu, Alpine Linux, or Node.js) and then layer your application on top. Docker Hub is a central repository where you can find pre-built images for various programming languages and services.

4. Docker Engine

  • What is Docker Engine: Docker Engine is the core component that runs containers. It is a client-server application with three major components:
    • Docker Daemon (dockerd): The background service that manages containers, ages, networks, and storage volumes.

      • Note: dockerd is not being used now a days as it’s obsolete. Now we are using ontainerd

    • Docker CLI (docker): The command-line interface that allows users to with the Docker Daemon.
    • REST API: Allows external programs to interact with Docker and manage containers.

5. Docker Compose

  • Docker Compose is a tool for defining and running multi-container Docker applications. With a simple YAML file (docker-compose.yml), you can configure services (e.g., web servers, databases) and their relationships, networks, and volumes.
    • Example docker-compose.yml:
    
    yaml
    version: '3'
    services:
    web:
    image: nginx
    ports:
    - "8080:80"
    db:
    image: mysql
    environment:
    MYSQL_ROOT_PASSWORD: example

6. Docker Registry:

  • A Docker registry is where Docker images are stored. Docker Hub is the most popular public registry, but organizations can also use private registries. Registries allow you to push and pull images to and from remote repositories.

7. Networking

  • Network Isolation Docker containers are isolated by default, and you can control how they communicate with each other and with the outside world. Docker networks enable this isolation and define how containers can communicate.

    • Default Network Types:

  • Bridge: Default network for standalone containers.
  • Host: The container shares the host's networking stack.
  • Overlay: Used for multi-host networking in Swarm mode (Docker's orchestration tool).
  • None: No network interface for the container.

8. Volumes

  • Volumes are used for persistent storage in Docker containers. By default, all data in a container is ephemeral (i.e., lost when the container is removed), but volumes allow you to persist data across container restarts or even container removals. Volumes can also be shared between containers.

9. Docker Swarm and Kubernetes

  • Docker Swarm is Docker's native orchestration tool for managing clusters of Docker hosts (or nodes). It allows you to deploy, manage, and scale applications across a cluster of machines.
  • Kubernetes is a more powerful container orchestration platform used to automate deployment, scaling, and operations of application containers. Kubernetes can be used with Docker (although Docker is being replaced as the default container runtime in some environments).

10. Benefits of Docker

  • Portability:Containers run the same regardless of the host system.
  • Efficiency: Containers are lightweight and start up quickly.
  • Isolation: Applications and services run in isolated environments, reducing dependency conflicts.
  • Version Control: Docker images are versioned, so you can easily roll back to previous versions.
  • CI/CD Integration: Docker integrates well with continuous integration and deployment pipelines.

11. Common Docker Commands

  • docker build: Builds a Docker image from a Dockerfile.
  • docker run: Runs a container from an image.
  • docker ps: Lists running containers.
  • docker stop: Stops a running container.
  • docker rm: Removes a stopped container.
  • docker pull: Pulls an image from a registry.
  • docker push: Pushes an image to a registry.

12. Use Cases for Docker

  • Microservices: Docker is often used in microservice architectures, where each service runs in its own container.
  • DevOps & CI/CD: Docker containers can be easily integrated into CI/CD pipelines to provide consistent environments for testing and production.
  • Cloud Deployment: Docker is widely used in cloud environments for application deployment and scaling.
  • Environment Consistency: Docker helps ensure that the environment for development, testing, and production are consistent.

Conclusion

  • Docker simplifies the process of developing, shipping, and running applications by using containers, making it easier to manage dependencies, scale applications, and maintain consistent environments across different stages of the software lifecycle.

Docker Basic Commands

Command Description
sudo apt install docker.io Install Docker on Ubuntu Linux system.
docker --version Check installed Docker version.
docker info Display detailed system-wide Docker information.
docker help Show help menu for Docker commands.
docker login Login to Docker Hub registry.
docker ps Show running containers.
docker ps -a Show all containers (running and stopped).
docker build -t image_name . Build a Docker image from Dockerfile.
docker pull image_name Download image from Docker Hub.
docker push image_name:tag Upload image to Docker Hub.

Docker Image Commands

Command Description
docker images List all Docker images available locally.
docker rmi image_name Remove a Docker image.
docker save image_name -o file.tar Save image to tar file.
docker load -i file.tar Load image from tar file.
docker search image_name Search image in Docker Hub.
docker image prune Remove unused Docker images.

Docker Container Commands

Command Description
docker run image_name Create and start a new container.
docker run -d --name container_name -p 8080:8080 image_name Run container in detached mode with port mapping.
docker stop container_name Stop a running container.
docker start container_name Start a stopped container.
docker rm container_name Remove a container.
docker logs container_name View container logs/output.
docker exec -it container_name /bin/bash Access container terminal interactively.

Docker Volume Commands

Command Description
docker volume ls List all Docker volumes.
docker volume create volume_name Create a new Docker volume.
docker run -v volume_name:/data image_name Attach volume to container.
docker volume inspect volume_name Show detailed volume information.
docker volume rm volume_name Remove a Docker volume.
docker volume prune Remove unused volumes.

Docker Networking Commands

Command Description
docker network ls List all available Docker networks.
docker network create network_name Create a new custom Docker network.
docker network connect network_name container_name Connect a container to a specific network.
docker network inspect network_name Display detailed information about a network.
docker network rm network_name Remove a Docker network.
docker network disconnect network_name container_name Disconnect a container from a network.

Docker Compose Commands

Command Description
docker compose up -d Start services in detached mode using docker-compose.yml.
docker compose down Stop and remove containers, networks, and volumes created by Compose.
docker compose ps List running services.
docker compose logs View output logs from services.
docker compose build Build or rebuild services.
docker compose stop Stop running services without removing them.
docker compose restart Restart services.
docker compose exec service_name bash Execute command inside a running service container.
docker compose pull Pull latest images for services.
docker compose up --scale service=3 Scale a service to multiple container instances.

Docker Cleanup Commands

Command Description
docker system prune -a Remove all unused containers, networks, images, and cache.
docker image prune -a Remove unused images (including dangling images).
docker container prune Remove all stopped containers.
docker volume prune Remove unused volumes.

Docker Security Commands

Command Description
docker scan image_name Scan Docker image for vulnerabilities.
docker history image_name Show image layer history.
docker inspect container_name Display detailed container configuration.
docker run --read-only image_name Run container with read-only file system for security.
