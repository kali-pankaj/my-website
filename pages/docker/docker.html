🧠 Best Practice Notes

🔹 Layering & caching: Order Dockerfile from less frequently changed → more frequently changed (dependencies first).
🔹 .dockerignore: Exclude node_modules, .git, secrets to keep images slim.
🔹 Use specific tags: Avoid `latest` in production; pin versions like `node:20-alpine`.

⚡ Performance Insight

✔️ Multi-stage builds reduce final image size by 70%+
✔️ Alpine-based images (5-10MB) vs slim variants
✔️ `docker system prune -a` — keep CI/CD clean
✔️ Use `--link` only on legacy, prefer user-defined networks.

🐳 Docker Swarm vs K8s

Swarm: simpler, Docker-native, easier for small clusters.
K8s: industry standard, auto-healing, rolling updates, complex.
💡 Use Compose for dev, then translate to K8s manifests.

🔐 Security Checklist

✔️ Run containers as non-root.
✔️ `--read-only` root filesystem + tmpfs for writeable areas.
✔️ Regularly `docker scan` / `docker scout`.
✔️ Use secrets manager instead of ENV for passwords.

📘 Docker Deep Dive · DevOps Foundations

  • What is Docker: Open platform for developing, shipping, and running apps inside lightweight containers.
  • Containers vs VMs: Containers share host OS kernel -> faster, smaller.
  • Key components: Docker Engine, containerd, Docker CLI, Docker Hub.
  • Dockerfiles define immutable images; Images are templates; Containers are running instances.
  • Volumes persist data; Networks isolate/interconnect containers.
  • Docker Compose orchestrates multi-container apps with YAML.
💡 Pro tip: Use `docker init` to generate Dockerfile + compose assets rapidly (experimental).

📦 Containers & Isolation

✔️ Lightweight, portable, share OS kernel.
✔️ Namespaces (PID, NET, MNT) + cgroups provide resource isolation.
✔️ Container = running instance of an image.

🏗️ Multi-Stage Dockerfile Blueprints (production ready)

🐳 Node + React
FROM node:20 AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
EXPOSE 80
☕ Spring Boot
FROM maven:3.9 AS build
WORKDIR /build
COPY pom.xml .
COPY src ./src
RUN mvn package

FROM eclipse-temurin:17-jre-alpine
COPY --from=build /build/target/*.jar app.jar
ENTRYPOINT ["java","-jar","app.jar"]
🐍 Python (uvicorn)
FROM python:3.12-slim AS builder
COPY requirements.txt .
RUN pip install --user -r requirements.txt

FROM python:3.12-alpine
WORKDIR /app
COPY --from=builder /root/.local /root/.local
COPY . .
CMD ["uvicorn", "main:app", "--host","0.0.0.0"]
🏎️ Go scratch
FROM golang:1.22 AS build
WORKDIR /src
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 go build -o app

FROM scratch
COPY --from=build /src/app /app
ENTRYPOINT ["/app"]
✨ Study note: multi-stage drastically reduces image size, no build tools in production runtime.
